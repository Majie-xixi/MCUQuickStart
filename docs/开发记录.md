# MCUQuickStart — 开发记录

## 项目目标

桌面 GUI 工具，选择芯片型号后一键生成 Keil5 打开即编译的嵌入式工程模板。

## 技术栈

- Python 3 + PyQt6（GUI）
- JSON 驱动（芯片数据库）
- 模板 + 占位符替换（代码和工程模板）
- 官方 SDK 文件实时拷贝（不打包库文件）

## 开发历程

### 第一阶段：核心架构（2026-05-19 下午）

**1. 项目骨架**
- 目录结构：`src/core/`（逻辑）+ `src/gui/`（界面）+ `src/resources/`（数据+模板）

## 关键设计原则

1. **数据驱动架构** — 加新芯片只需新增 JSON + 模板，不改 Python
2. **SDK 文件实时拷贝** — 工具不打包固件库，生成时从官方 SDK 目录获取最新版
3. **芯片专属文件从 SDK Template 取** — `*_it.c`、`systick.c/h`、`main.h` 等跟着 SDK 版本走
4. **单一 SDK Root** — 所有芯片固件包放同一目录，工具自动匹配
5. **每芯片独立配置** — `define`、`flash_driver`、`startup`、`device`、`fwlib_exclude` 都在 JSON 中显式声明
6. **占位符 > 硬编码** — 所有芯片差异通过 `{{PLACEHOLDER}}` 由模板引擎填充

### 第二阶段：FreeRTOS 可选库集成（2026-05-21，feature/freertos-integration）

**目标**：工具支持在生成工程时可选集成 FreeRTOS（后续扩展 LVGL），首批从 GD32F10x 起步。

**1. 调研与方案设计**
- FreeRTOS V9 vs V10 port 层对比：ARM Cortex-M 的 RVDS port 层功能一致，V10 主要变化是许可证（GPL→MIT）+ 新增 `configSYSTICK_CLOCK_HZ` 支持
- SysTick 冲突分析：裸机 delay 模块使用 SysTick 一次性模式（LOAD→启动→轮询→停止），FreeRTOS 需要 SysTick 周期性 1ms 中断。解决方案：FreeRTOS 模式使用 DWT 周期计数器做 us 延迟，不碰 SysTick
- 参考用户已有移植项目 `AF1000_GD32_FREERTOS` 的实践：共享 SysTick_Handler（调度器启动后才转发到 xPortSysTickHandler）、TIMER3 做 us 延迟（工具中用 DWT 替代）、FreeRTOSConfig.h 的中断优先级映射

**2. SysTick 处理方案**
- `SysTick_Handler` 采用共享模式：`if (xTaskGetSchedulerState() != taskSCHEDULER_NOT_STARTED)` → `xPortSysTickHandler()`
- `delay_us()` FreeRTOS 模式使用 DWT 周期计数器（`DWT->CYCCNT`），不操作 SysTick 寄存器
- `delay_ms()` FreeRTOS 模式：调度器运行后走 `vTaskDelay()`，启动前走 DWT 忙等
- `delay_init()` FreeRTOS 模式：只初始化 DWT，不配置 SysTick（FreeRTOS 的 `vPortSetupTimerInterrupt()` 接管）
- `sysconfig.h` 新增 `SYSTEM_SUPPORT_OS = 2`（FreeRTOS 模式），保持 0=裸机、1=UCOS 向后兼容

**3. 新增文件清单**
- `templates/freertos/FreeRTOSConfig.h` — V10.3.1 配置模板，`{{SYSTICK_CLOCK_HZ_DEFINE}}` 占位符（F1 系列分频 /8）
- `templates/freertos/gd32f10x_it.c` — FreeRTOS 版中断处理，SysTick_Handler 转发到 xPortSysTickHandler
- `templates/gd32f10x/freertos/{empty,led,uart}/main.c` — FreeRTOS 版 main.c，xTaskCreate + vTaskStartScheduler
- `gd32f10x.json` 新增 `optional_libs.freertos` 配置：sdk_subdir、core_files、port_map、heap、systick_clock_div

**4. project_generator.py 改造**
- `generate()` 新增 `optional_libs` 参数（`list[str] | None`）
- `_setup_freertos()`：从 SDK 拷贝 FreeRTOS 内核/port/heap 源文件、渲染 FreeRTOSConfig.h、覆盖 it.c
- `_code_vars()` 根据 `use_freertos` 设置 `SYSTEM_SUPPORT_OS` 和 `SYSTICK_CLOCK_HZ_DEFINE`
- `_freertos_groups_xml()`：生成 FreeRTOS_CORE + FreeRTOS_PORT 的 uvprojx Group XML
- `_freertos_includes()`：生成 FreeRTOS 额外的 include 路径
- `_build_uvprojx_vars()` 新增 `FREERTOS_GROUPS`、`FREERTOS_INCLUDES` 模板变量
- uvprojx_template.xml 新增 `{{FREERTOS_GROUPS}}` 和 `{{FREERTOS_INCLUDES}}` 占位符

**5. GUI 改造**
- 新增"可选库"QGroupBox + FreeRTOS QCheckBox
- `_on_generate()` 根据勾选状态传递 `optional_libs=["freertos"]`

## 经验教训

- **芯片参数必须查官方 datasheet 核实**，不能从相似型号照抄
- **用户反馈要验证后再改**，用户说的不一定全对（如 F470ZET6 不是 512K）
- **commit 前自查**：include 链完整性、宏定义生效、警告清零
